Extract Google Analytics data using Reporting API - Part 1

2020-09-25
tutorial
286 words 2 mins read

Contents

This is a two part tutorial to teach how you can extract google analytics data using Reporting API.

In the first part, we will be covering some prerequisite setup which will allow us to leverage Reporting API.

  1. Create a new project by logging into your google API console

  2. Click on Credentials under side menu, select CONFIGURE CONSENT SCREEN

  3. Under User Type select External

    • While configuring OAuth consent screen, you just need to profide Application name and support email.

  4. Go back to Credentials screen and click on CREATE CREDENTIALS, choose Oauth Client ID from the menu.

  5. Choose Application type to be Web Application

    • Give a name to your Oauth client
    • Add https://developers.google.com/oauthplayground as Authorized redirect URI and click on CREATE

  6. Once created you will be presented with a unique Client ID and Secret (you don’t have to make a note of it, you can always access it later)

  7. Using Client ID and secret we will now extract a refresh token from Google OAuth playground:

    • Go to Google Oauth playground
    • Click on the gear icon on top right hand side corner (step 1 in image)
    • Check the option Use your own OAuth credentials (step 2 in image)
    • Enter Client ID and secret from # 6 (step 3 in image)
    • In Input your own scope add https://www.googleapis.com/auth/analytics.readonly and hit Authorize APIs (step 4 in image)

  • Follow prompts to provide consent.
  • After APIs are authorized, click Exchange authorization code for tokens
  • This will generate Refresh token, copy and keep this refresh token securely

We have now completed the setup and have the three important components to authenticate and extract data from Google Analytics reporting API:

  1. Client ID
  2. Client Secret
  3. Refresh Token

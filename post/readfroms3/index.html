Copy data from Amazon S3 bucket using Azure Data Factory

2020-12-01
tutorial
242 words 2 mins read

Contents

  • Solution
    • Linked Service
    • Source Dataset
    • Copy Pipeline

Objective: Copy a file from AWS S3 bucket using Azure Data Factory

I tried to follow the official Microsoft documentation, but kept getting this error message when creating Source dataset:

Failed to load

The file operation is failed. A webException with status TrustFailure was thrown. The underlying connection was closed: Could not establish trust relationship for the SSL/TLS secure channel. The remote certificate is invalid according to the validation procedure.

Solution

Linked Service

  • Authentication type = Access key
  • Access Key ID = use your access key id
  • Secret Access Key = use your secret access key (Best Practice is to save this in Azure Key vault and use Secret name to access it)
  • Service URL = http://s3.us-east-1.amazonaws.com/

This is where I had to do a lot of back and forth to figure out what should be the correct service URL. I found this article on Amazon’s website pretty helpful

  • Test Connection = To linked service

Source Dataset

  • Linked service = select Linked service created earlier
  • File path = bucket name/directory path/filename

If you are not sure about directory path, use this aws cli command to figure it out

1

aws s3 ls s3://bucketname --recursive --human-readable --summarize

Copy Pipeline

  • Source Dataset = select dataset created earlier
  • File path type = File path in dataset

You can now click on Preview Data to see that you are now able to successfully read the file.

Rest of the flow should be simple, hope this helps.
